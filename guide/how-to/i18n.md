# i18n

Internationalisation (i18n) provides a way for you to customise your shop to accommodate users from different countries.

## Locales

The i18n configuration affects language and currency. It's important that the app's configuration matches the settings of the system [you connect to](middleware.md). Otherwise, you'll receive incorrect data or data in the wrong language.

- The locale settings have to match the PlentyONE system settings.
- The `fallbackLocale` has to match the system default language.
- The `defaultLocale` has to match the system default language.

To update any part of the configuration, modify `i18n.config.ts`.

::: info
By default, the app ships with English as the default locale and German as a second locale.
:::

### Adding a new locale

To add a new locale:

1. Add your new locale code (e.g., `es`) to the `allLocales` array in `getLocales()` in `i18n.config.ts`.
2. Create `es.json` (translations) in `apps/web/app/lang/`.
3. Add the appropriate currency format for your locale in `vueI18n.config.ts` under `numberFormats`.
4. Make sure the PlentyONE system settings are updated to support the new locale.

#### Example: Dynamic locale and currency config

`i18n.config.ts`:

```ts
export const getLocales = (): LocaleObject[] => [
  { code: "en", file: "en.json" },
  { code: "es", file: "es.json" },
  // ...other locales
];

export const nuxtI18nOptions: NuxtI18nOptions = {
  locales: getLocales(),
  defaultLocale: getDefaultLocale(),
  langDir: "../app/lang",
  strategy: "prefix_and_default",
  vueI18n: "~/configuration/vueI18n.config.ts",
  detectBrowserLanguage: false,
  lazy: true,
};
```

`vueI18n.config.ts`:

```ts
export default {
  fallbackLocale: process.env.DEFAULTLANGUAGE ?? "en",
  numberFormats: {
    en: {
      currency: {
        style: "currency",
        currency: "GBP",
        currencyDisplay: "symbol",
      },
    },
    es: {
      currency: {
        style: "currency",
        currency: "EUR",
        currencyDisplay: "symbol",
      },
    },
    // ...other locales
  },
};
```

#### Supported locales

`bg`, `cs`, `da`, `de`, `en`, `es`, `et`, `fi`, `fr`, `ga`, `hr`, `hu`, `it`, `lt`, `lv`, `nl`, `no`, `pl`, `pt`, `ro`, `ru`, `sk`, `sv`, `tr`, `vi`, `zh`

#### Configuration notes

- The locale settings must match the PlentyONE system settings
- The `fallbackLocale` and `defaultLocale` are determined dynamically from the `DEFAULTLANGUAGE` environment variable (falling back to English if not set or invalid)
- All locale files are located in `apps/web/app/lang/` and must be present as `<locale>.json` (translations)
- Locales are dynamically loaded using the `getLocales()` function in `apps/web/app/configuration/i18n.config.ts`
- To update any part of the configuration, modify `i18n.config.ts` and `vueI18n.config.ts`

## Translations

The localization function `t(...)` is now **auto-imported** and **auto-completed** in all Vue components. You no longer need to manually import or inject `t`—just use it directly in your templates and `<script setup>` blocks.

### Usage Example

**Before (manual import/injection):**

```vue
<template>
  <div>{{ t("your.translation.key") }}</div>
</template>

<script setup lang="ts">
const { t } = useI18n();
</script>
```

**Now (auto-imported):**

```vue
<template>
  <div>{{ t("your.translation.key") }}</div>
</template>

<script setup lang="ts">
// No import needed!
</script>
```

**Key features:**

- No import required
- Auto-completion works in supported IDEs (e.g., VS Code)
- Works in both template and `<script setup>`

### Translation Key Structure

All translation keys must follow this format:

```
key.key.key.key.value
```

**Rules:**

- Keys can have up to **5 levels** (e.g., `checkout.address.form.label`)
- **Every key must have at least one parent** (e.g., `account.name` is valid, `name` is not)
- Flat keys (single level, like `edit`) are **not allowed**

**Examples:**

- `account.settings.profile.edit`
- `checkout.address.form.label`
- `product.details.price.value`

This structure ensures consistency and clarity in all translation files.

## Localization Editor

The shop now includes a **localization editor** that allows users to edit translations directly per language. This editor provides a user-friendly interface for managing all customer-facing translations without needing to manually edit JSON files.

### Editor UI Translations

::: warning
**Important:** Editor UI keys (used for the editor interface itself, settings panels, toolbar components, etc.) are **not included** in the localization editor and are **not available for modification** by users.
:::
These translations are:

- Isolated from the main translation system
- Stored separately in `apps/web/app/utils/editorTranslations.en.json` and `editorTranslations.de.json`
- Managed via the `getEditorUITranslation()` helper function
- Only modifiable by developers through code changes

This separation ensures that the editor interface remains stable and consistent, while giving users full control over their shop's customer-facing content.

#### Developer Guide: Adding Editor UI Translations

When adding new editor UI translations (for settings panels, toolbar components, etc.), follow these steps:

1. **Add translation keys to both JSON files:**
   - `apps/web/app/utils/editorTranslations.en.json`
   - `apps/web/app/utils/editorTranslations.de.json`

2. **Use the `getEditorUITranslation()` helper in your components:**

   ```vue
   <template>
     <div>{{ getEditorUITranslation("your-key-name") }}</div>
   </template>
   ```

3. **For translations with parameters:**

   ```vue
   <template>
     <div>{{ getEditorUITranslation("deleteSuccess", { pageName, id }) }}</div>
   </template>
   ```

4. **Key structure:**
   - Use kebab-case for keys (e.g., `branding-and-design`, `custom-canonical-URL-for-item-pages`)
   - Keys can be simple strings without dots (unlike the main i18n system)
   - Examples: `toolbarSuccess`, `deleteError`, `filter.prices.price_asc`

5. **Testing:**
   - Run the test suite to ensure all keys are present in both language files
   - Verify parameter placeholders match between languages
   - Test: `npm run test apps/web/app/utils/__tests__/editorTranslationsHelper.spec.ts`

#### What Changed in the Editor Translation System

**Removed:**

- All `lang.json` files from settings folders
- `$registerMessages` calls from View components
- `<i18n>` blocks from settings components

**Added:**

- Two centralized JSON files for all editor UI translations
- `getEditorUITranslation()` helper function replacing `t()` and `$dynamicEditorTranslation()`

**Benefits:**

- Editor translations are completely isolated from customer-facing translations
- No risk of editor UI keys appearing in the localization editor
- Centralized management of all editor UI translations
- Consistent naming and structure across the editor interface

## Migration Guide

### Shop version v2.8.0

Starting with shop version v2.8.0 we introduced the new localization editor, in order to be able to translate all keys you need to use our custom auto-imported `t` function.

#### Migrating to Auto-Imported `t`

1. **Remove** any manual imports or injections of `t` in your components, [example](/guide/how-to/i18n.md#usage-example)
2. Use `t('key.path')` directly in your templates and scripts
3. Make sure your IDE is updated for best auto-completion support

#### Compatibility

Auto-imported `t` is available in:

- **plentyshop-pwa**: v2.8.0
- **@plentymarkets/shop-core**: ^1.14.0

::: warning
⚠️ If you are using an older version, you must continue to import or inject the `t` function manually.
:::

## FAQ

**Q: Do I need to import `t` in every file?**
A: No. `t` is globally available in all Vue components.

**Q: Does this work in both `<template>` and `<script setup>`?**
A: Yes, you can use `t` in both places without import.

**Q: What if my IDE doesn't autocomplete?**
A: Make sure you are using a compatible version and your IDE is up to date.

**Q: What's the difference between `t()` and `getEditorUITranslation()`?**
A: `t()` is for customer-facing translations (editable in the localization editor), while `getEditorUITranslation()` is for editor interface translations (developer-only).
